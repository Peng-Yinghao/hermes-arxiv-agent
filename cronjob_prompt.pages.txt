## 任务：每天自动监控 arXiv 论文并推送微信

### 第零步：安装运行时依赖
在项目根目录执行（仅首次需安装，后续跳过）：
```bash
cd /path/to/hermes-arxiv-agent
pip install --break-system-packages --quiet openpyxl requests pdfplumber 2>/dev/null || pip install --break-system-packages --target=.deps --quiet openpyxl requests pdfplumber
export PYTHONPATH="/path/to/hermes-arxiv-agent/.deps:${PYTHONPATH:-}"
```

### 第一步：执行 Python 脚本
cd 到项目根目录后，用 python3 运行：
```bash
cd /path/to/hermes-arxiv-agent && python3 monitor.py
```

### 第二步：判断是否有新论文
运行完毕后检查脚本输出：
- 如果包含 `No new papers`，直接回复：`✅ 今日（YYYY-MM-DD）未发现新的论文。`
- 如果有 `LLM_SUMMARIZATION_REQUIRED` 部分，进入第三步。

### 第三步：生成中文总结 summary_cn
读取 `/path/to/hermes-arxiv-agent/new_papers.json`，该文件包含 `new_count`、`pending_count`、`new_papers` 和 `papers_to_process`。

其中：
- `new_count` 表示本次新抓取的论文数
- `pending_count` 表示当前仍需要 agent 补全的论文数
- 优先以 `papers_to_process` 作为本次必须处理的论文列表；若不存在，再使用 `new_papers`

对待处理列表中的每篇论文，基于 abstract 生成中文总结：
- 90-150 个中文字符
- 必须覆盖：方法核心、主要贡献、关键结果
- 禁止模板化空话，禁止仅写"提出新型方法"等泛化句
- 只输出一段自然中文，不使用分点
- 质量自检：少于 90 字或与其他摘要高度雷同时重写

### 第四步：更新 Excel
将 summary_cn 回填到 `/path/to/hermes-arxiv-agent/papers_record.xlsx`（按 arxiv_id 匹配）。
affiliations 列保留为空即可，本 workflow 不需要提取作者单位。

### 第五步：论文分类（token 优化）
在项目根目录执行：
```bash
cd /path/to/hermes-arxiv-agent && python3 classify_papers.py --prepare
```

读取 `classify_input.json`，基于论文标题和摘要（不读全文 PDF），按三维度批量分类：
- **应用场景**: 个人助手 / 代码Agent / Web Agent / 具身Agent / 多智能体模拟 / 通用其他
- **记忆形式**: 文本 / 向量 / 结构化 / 参数化 / 潜变量 / 混合其他
- **记忆类型**: 上下文摘要型 / 检索型 / 反思经验型 / 分层管理型 / 结构化图谱型 / 参数潜变量型 / 其他

将分类结果写入 `classify_output.json`，然后执行：
```bash
cd /path/to/hermes-arxiv-agent && python3 classify_papers.py --apply
```

该步骤只处理未分类的新论文，已分类的自动跳过，token 消耗极低（仅标题+摘要）。

### 第六步：更新网站数据
在项目根目录执行：
```bash
cd /path/to/hermes-arxiv-agent && python3 viewer/build_data.py
```

确认 `viewer/papers_data.json` 已根据最新 Excel 重建，包含刚回填的 summary_cn。

### 第七步：生成 Markdown 文档
在项目根目录执行：
```bash
cd /path/to/hermes-arxiv-agent && python3 build_markdown.py
```

确认 `markdown/` 目录下已生成每篇论文的 `.md` 文件和 `README.md` 索引。

### 第八步：生成全文 Markdown（arxiv2md）
在项目根目录执行：
```bash
cd /path/to/hermes-arxiv-agent && python3 build_markdown_full.py
```

该脚本使用 arxiv2md 从 arXiv HTML 版本下载论文全文 Markdown，存入 `markdown_full/` 目录（每篇论文一个 `.md` 文件，包含完整章节结构）。对于没有 HTML 版本的旧论文会自动跳过。

### 第九步：同步待处理队列
在项目根目录执行：
```bash
cd /path/to/hermes-arxiv-agent && python3 monitor.py --sync-pending-state
```

要求：
- 该步骤必须在 Excel 回填完成后执行
- 该步骤会根据最新 Excel 状态重建 `pending_llm_ids.txt`
- 若全部论文都已完成中文总结，应自动清空 `pending_llm_ids.txt`
- 该步骤还会同步刷新 `new_papers.json`，避免 agent 重试时继续读取旧的待处理论文列表

### 第十步：发布静态网站
在项目根目录执行：
```bash
cd /path/to/hermes-arxiv-agent && bash scripts/publish_viewer.sh
```

要求：
- 仅提交站点相关文件，不要提交 `papers_record.xlsx`、`papers/`、`viewer/favorites.json`
- 如果 `viewer/papers_data.json` 没有变化，可以跳过提交和 push
- push 到你自己的 GitHub fork 后会触发 GitHub Actions 更新 GitHub Pages
- 当前仓库必须是你自己的 fork，或 `PUBLISH_REMOTE` 必须明确指向你自己的可写远端

### 第十一步：构建微信 Markdown 消息
输出格式：
```
📚 **论文日报** | YYYY-MM-DD
共发现 **N** 篇新论文
[每篇：标题、arXiv ID、日期、作者、PDF 链接、90-150 字中文摘要]
PDF 已下载至 papers/，记录已更新至 papers_record.xlsx，网站数据已更新至 viewer/papers_data.json。
在线查看：https://peng-yinghao.github.io/hermes-arxiv-agent/
```
**重要：最终回复只输出微信消息文本，不要输出其他说明。**
