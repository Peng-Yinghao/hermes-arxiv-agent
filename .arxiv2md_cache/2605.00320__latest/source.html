VitaLLM: A Versatile and Tiny Accelerator for Mixed-Precision LLM Inference on Edge Devices

Zi-Wei Lin, and Tian-Sheuan Chang
Abstract

We present VitaLLM, a mixed-precision accelerator that enables ternary-weight large language models to run efficiently on edge devices. The design combines two compute cores—a multiplier-free TINT core for ternary–INT projections and a BoothFlex core that reuses a radix-4 Booth datapath for both INT8×\timesINT8 attention and ternary–INT—sustaining utilization without duplicating arrays. A predictive sparse attention mechanism employs a leading-one (LO) surrogate with a comparison-free top-KK selector to prune key/value (KV) fetches by roughly 1−K/M1{-}K/M for MM cached tokens, confining exact attention to KK candidates. System-level integration uses head-level pipelining and an absmax-based quantization barrier to standardize cross-core interfaces and overlap nonlinear reductions with linear tiles. A 16 nm silicon prototype at 1 GHz/0.8 V achieves 72.46 tokens/s in decode and 0.88 s prefill (64 tokens) within 0.214 mm² and 120 KB on-chip memory, while reducing KV traffic and improving utilization in ablations. These results demonstrate practical BitNet b1.58 (3B) inference on edge-class platforms and provide a compact blueprint for future mixed-precision LLM accelerators.

I Introduction

Large language models (LLMs) have revolutionized natural language understanding, but their growing scale stresses computation, memory, and energy—especially on edge platforms that require always-on, low-latency inference under tight power and capacity budgets. Meeting these constraints demands model compression and hardware co-design, particularly below 8-bit precision.

BitNet b1.58 [4, 3] advances this goal by using ternary weights {−1,0,+1}\{-1,0,+1\} with 8-bit activations. Trained with quantization-aware methods, BitNet b1.58 achieves competitive accuracy from the 3B-parameter scale while reducing memory (up to 7×7\times reduction) and computation. In BitNet, BitLinear layers replace conventional linear projections in both attention and feedforward modules (see Fig. 1).

Specialized hardware is required to realize these algorithmic gains in practice. Prior ternary LLM accelerators include Slim-Llama (ASIC) [1], TerEffic (FPGA) [7] and TeLLMe (edge FPGA) [5]. Nonetheless, three challenges remain for edge deployment: (i) poor array utilization between ternary and INT8 attention, (ii) high KV-cache bandwidth from random access, and (iii) complex mixed-precision/nonlinearity interfaces.

We present VitaLLM, an edge-oriented mixed-precision accelerator that addresses these challenges with three contributions:

  • •

    Dual-core compute. A multiplier-free TINT core for all ternary–INT projections and a BoothFlex core that executes both INT8×\timesINT8 (attention) and ternary–INT on a shared radix-4 Booth datapath, improving utilization without duplicating arrays.

  • •

    Predictive sparse attention. A leading-one (LO) surrogate with a comparison-free top-KK selector that reduces KV fetches by roughly 1−K/M1-\!K/M (for MM cached tokens) and confines exact attention to the KK candidates—no retraining required.

  • •

    System integration. Head-level pipelining that consumes Q/K/V immediately per head, plus an absmax-based quantization barrier that decouples nonlinear reductions from scaling to provide a uniform (integer vector, single scale) interface across cores.

The remainder of this paper is organized as follows: Section II describes the TINT and BoothFlex microarchitectures. Section III presents system integration. Section IV reports implementation and measurements. Section V concludes.

II Compute Core Microarchitectures

We employ two multiplier-light compute arrays that share on-chip buffers and a common output-stationary (OS) dataflow. The TINT core handles all ternary–INT projections, while BoothFlex reuses a radix-4 Booth pipeline to execute both INT8×\timesINT8 attention and ternary–INT without duplicating arrays. This co-design sustains hardware utilization while containing area and memory traffic.

II-A TINT Core

TINT (Fig. 2) accelerates ternary–INT dot products with a multiplier-free 8×88{\times}8 PE array and OS mapping. Each PE receives an activation aa and a ternary weight w∈{−1,0,+1}w\!\in\!\{-1,0,+1\}, and updates a local accumulator using a lightweight selector:

y←y+sel​(w,a),sel​(w,a)∈{0,+a,−a}.y\leftarrow y+\mathrm{sel}(w,a),\quad\mathrm{sel}(w,a)\in\{0,\,+a,\,-a\}.

The selector is implemented as a small mux with sign inversion driven by a ternary decoder, avoiding general-purpose multipliers. Per cycle, the array broadcasts eight INT activations across columns while all 8×88{\times}8 PEs consume 64 ternary weights, sustaining 64 ternary-INT select-accumulates per cycle. Outputs are emitted in fixed-size tiles while partial sums remain on-chip, minimizing buffer traffic and toggling. A compact control path streams packed ternary codes and INT activations, enabling high throughput for Q/K/V, WOW_{O}, and FFN projections on a ternary-arithmetic datapath.

II-B BoothFlex Core

Transformer attention requires INT8×\timesINT8 products in addition to ternary–INT. BoothFlex reuses one radix-4 Booth datapath for both modes (Fig. 3). The recoder scans overlapping 3-bit windows {y2​i+1,y2​i,y2​i−1}\{y_{2i+1},y_{2i},y_{2i-1}\} (with y−1=0y_{-1}{=}0) to produce ri∈{−2,−1,0,+1,+2}r_{i}\!\in\!\{-2,-1,0,+1,+2\}. For a bb-bit multiplier, the iteration count is N=⌈b+12⌉N=\left\lceil\frac{b+1}{2}\right\rceil. Thus, INT8×\timesINT8 requires b=8⇒N=5b{=}8\Rightarrow N{=}5 cycles. For ternary–INT, we zero-pad the 2-bit ternary code to one 3-bit window (−1:11⇒110, 0:00⇒000,+1:01⇒010-1{:}11\!\Rightarrow\!110,\;0{:}00\!\Rightarrow\!000,\;+1{:}01\!\Rightarrow\!010), yielding N=1N{=}1. Thus ternary and INT8 operands share the same Booth pipeline without extra arithmetic units.

At the PE level, computation is bit-serial: one Booth iteration per cycle with row accumulators updated as

PSi←(PSi−1≪2)+PPi,\mathrm{PS}_{i}\leftarrow(\mathrm{PS}_{i-1}\ll 2)+\mathrm{PP}_{i},

where PPi\mathrm{PP}_{i} sums partial products selected by rir_{i}. INT8×\timesINT8 spans five cycles; ternary–INT collapses to a single cycle. As with TINT, OS mapping keeps partial sums local and outputs are emitted in 8-element chunks. This schedule trades small latency for sizable area/power savings, while the head-level pipeline (Sec. III-B) later maintains high utilization.

III System Integration and Scheduling

VitaLLM integrates the TINT and BoothFlex cores under a shared output-stationary (OS) schedule and lightweight control network. The schedule keeps arrays utilized by (i) pruning attention candidates before costly KV fetch, (ii) pipelining work at the head granularity, and (iii) standardizing mixed-precision interfaces via an absmax-based quantization barrier.

III-A Predictive Sparse Attention

We reduce key/value (KV) traffic and attention MACs by screening candidate keys using a multiplier-free Leading-One Prediction (LOP) surrogate that correlates with the true dot product.

Let q,k∈ℤdq,k\!\in\!\mathbb{Z}^{d} be INT8 query and key vectors. Define

LO​(x)=⌊log2⁡|x|⌋,sgn⁡(x)∈{−1,+1}.\mathrm{LO}(x)=\Big\lfloor\log_{2}|x|\Big\rfloor,\qquad\operatorname{sgn}(x)\in\{-1,+1\}.

The surrogate score is

s^​(q,k)=∑i=1dsgn⁡(qi)​sgn⁡(ki)​ 2LO​(qi)+LO​(ki).\hat{s}(q,k)=\sum_{i=1}^{d}\operatorname{sgn}(q_{i})\operatorname{sgn}(k_{i})\;2^{\,\mathrm{LO}(q_{i})+\mathrm{LO}(k_{i})}. (1)

Each term reduces to a barrel-shifted 1, enabling a compact ExpAdd PE that streams (q,k)(q,k) once to accumulate (1). A streaming top-KK (or threshold) selector emits only the highest-scoring indices for exact attention, while all others are skipped (Fig. 4). This maintains attention quality and converts random KV probes into short, contiguous reads that the memory system can serve efficiently.

To achieve fixed-KK selection with predictable throughput and small area, we adopt a comparison-free, kk-degree parallel design inspired by [6]. Scores are bucketized into discrete ranges; a high-to-low prefix scan locates the cut bin where the cumulative count first reaches KK; then kk-wide priority encoders emit up to kk indices per cycle, avoiding wide comparator trees and matching the streaming access pattern to cached KK.

At the system level: (i) KV traffic. Only keys in the kept set 𝒞\mathcal{C} are fetched for exact q​K𝒞⊤qK_{\mathcal{C}}^{\top} and the subsequent 𝐒𝐕\mathbf{SV}, so average KV fetches scale with KK rather than the cache length MM. (ii) Arithmetic. Computing (1) replaces dot-product multiplications with shifts and additions, and exact INT8×\timesINT8 attention is performed only for 𝒞\mathcal{C}, reducing compute roughly with K/MK/M and simplifying the head-interleaved pipeline in Sec. III-B, all without retraining.

III-B Head-Level Pipelining

To balance compute and memory traffic across the two cores, we organize multi-head attention as a two-stage stream at head granularity (Fig. 5). A conventional schedule computes Q/K/V for all heads and materializes {𝐐h,𝐊h,𝐕h}h=0H−1\{\mathbf{Q}_{h},\mathbf{K}_{h},\mathbf{V}_{h}\}_{h=0}^{H-1} before starting attention—incurring an extra round of writes and later re-reads. Instead, we use a one-head–offset pipeline: as soon as head hh finishes Q/K/V on TINT, the same head immediately enters attention on BoothFlex using LOP-derived top-KK indices (Sec. III-A), while TINT advances to Q/K/V for head h+1h{+}1. Q/K/V are consumed promptly, avoiding bulk intermediate storage and reducing inter-stage traffic. See Fig. 5.

This schedule also exploits BoothFlex dual mode. During MHA, BoothFlex runs INT8×\timesINT8 for 𝐐𝐊⊤\mathbf{QK}^{\top} and 𝐒𝐕\mathbf{SV}. After all heads finish and outputs are concatenated, both cores switch to ternary–INT and cooperate on WOW_{O} and FFN (up, gate, down). The result: no Q/K/V buffering, attention begins as soon as each head is ready, and utilization stays high as BoothFlex switches from INT8×\timesINT8 to ternary–INT after MHA.

III-C Nonlinear Decoupling and Absmax-Based Quantization Barrier

Unlike fully fused flows that perform 𝐐𝐊⊤→softmax→𝐒𝐕\mathbf{QK}^{\top}\!\rightarrow\!\mathrm{softmax}\!\rightarrow\!\mathbf{S}\mathbf{V} inside one array [2], VitaLLM crosses a quantized interface between two heterogeneous cores. To keep the stream, we decouple data-dependent reductions from scaling/normalization. As the upstream linear kernel (e.g., QK, projections) emits tiles, a small scaling unit multiplies each tile by its activation scale and feeds a parallel reduction pipeline: running max and sum of exponentials for softmax, or sum of squares for RMSNorm. When the final tile of the vector arrives, these reductions are complete; we then perform the normalization over the full vector, compute a single per-vector scale, quantize once, and forward the (integer vector, scale) pair. The downstream linear kernel runs entirely in the integer domain and needs only one output-side dequantization before its following nonlinearity. (See Fig. 6.)

For our 3B BitNet b1.58 setting, inter-stage scaling uses per-vector absmax α=maxi⁡|xi|\alpha=\max_{i}|x_{i}|. Because α\alpha itself is a vector-wide reduction, it naturally serves as the synchronization barrier: quantization can occur only after the reduction completes. We exploit this to modularize the flow—reductions overlap with vector production, quantization fires once per vector, and the same reduction module is time-multiplexed across operators. The result is a latency-hidden, vector-granular pipeline that fits our head-level streaming schedule without feedback paths or multi-ported SRAMs (Fig. 6).

IV Implementation and Experimental Evaluation

IV-A Overall Architecture

Fig. 7 shows the top level. The design instantiates three TINT cores, one BoothFlex core, a lightweight LOP unit, a nonlinear block, and on-chip buffers. Weights and the KV cache reside in off-chip DDR; tiles are DMA mapped to on-chip buffers and executed with an output-stationary mapping so partial sums stay in local accumulators.

LOP sits on the fast path between TINT and memory. When a head’s Q/K/V tiles finish on TINT, the query features (LO,sgn)(\mathrm{LO},\mathrm{sgn}) are compared to cached key features to form a top-KK set; only those candidate blocks are requested from the KV cache for exact attention. This reduces KV traffic by roughly 1−K/M1{-}K/M for MM cached tokens, and confines INT8×\timesINT8 work to the selected keys (Sec. III-A).

Buffers are bank-shared with fixed ports (TINT→\rightarrowA, BoothFlex→\rightarrowB) and a small credit counter; a tile dispatcher issues ready tiles. After MHA, both cores switch to ternary–INT and cooperatively process WOW_{O}/FFN at the same granularity.

The RTL targets 16 nm CMOS at 1 GHz/0.8 V with clock and operand gating to suppress toggling. Detailed PPA, throughput, and ablations are reported in Sec. IV-B.

IV-B Experimental Results

We evaluate VitaLLM on the silicon prototype in Sec. IV-A (TSMC 16 nm, 1 GHz, 0.8 V).

End-to-end comparison. Prior ternary accelerators make different trade-offs: ASIC Slim-Llama emphasizes prefill with heavy compute, which can leave the hardware underutilized in bandwidth-limited decode [1]; FPGA designs (TerEffic, TeLLMe) spread resources more evenly across prefill/decode [7, 5]. VitaLLM brings this balanced approach to ASIC and further sustains utilization via head-level streaming plus an LOP-driven top-KK gate that trims KV fetches before exact attention (Sec. III-A). For edge settings with moderate sequence lengths, modest prefill parallelism is acceptable and keeps both cores busy.

V Conclusion

We presented VitaLLM, a mixed-precision edge accelerator that couples a multiplier-free TINT core for ternary–INT projections with a BoothFlex core that runs both INT8×\timesINT8 (attention) and ternary–INT (projections) on a shared Booth datapath, lifting utilization without a second array. A leading-one predictor with a comparison-free top-KK selector prunes KV fetches by roughly 1−K/M1{-}K/M (for MM cached tokens) and restricts exact attention to only KK candidates, with no retraining. Per-head streaming consumes Q/K/V immediately, and an absmax-based barrier decouples nonlinear reductions (softmax/RMSNorm) from scaling, yielding a uniform (integer vector, single scale) interface across cores.

Our 16 nm, 1 GHz/0.8 V prototype delivers 72.46 tokens/s decode within 0.214 mm2 at 59.12 mW. In ablations, LOP reduces KV traffic by 54.86×\times and speeds up MHA by 26.31%; head-level pipelining adds 54.31%; BoothFlex utilization rises to 69.20%, yielding an overall +38.17% throughput increase when combined with HLP. These results indicate that VitaLLM enables practical 3B BitNet b1.58 inference on edge devices and offer a blueprint for efficient mixed-precision LLM accelerators.
